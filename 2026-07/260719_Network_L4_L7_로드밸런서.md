 [Network] L4/L7 로드밸런서 및 방화벽 동작 원리

작성일: 2026.07.19
주제: 안전하고 확장성 있는 엣지-클라우드 아키텍처 설계를 위한 트래픽 분산 기법 학습

1. 로드밸런싱(Load Balancing)의 이해

트래픽이 폭증할 때 단일 서버가 받는 부하를 여러 대의 서버(Scale-out)로 적절히 분산시키는 필수적인 네트워크 기술. 시스템의 가용성과 응답 속도를 높이는 데 핵심적인 역할을 한다.

2. L4 로드밸런서 (Transport Layer)

동작 방식: IP 주소와 TCP/UDP 포트 정보를 바탕으로 트래픽을 분산한다.

장점: 패킷의 내용을 깊게 까보지 않기 때문에 속도가 매우 빠르고 효율적이며 리소스 소모가 적다.

단점: 패킷 내부의 애플리케이션 데이터(HTTP 헤더 등)를 볼 수 없어 정교한 라우팅은 불가능하다.

3. L7 로드밸런서 (Application Layer)

동작 방식: HTTP 헤더, 쿠키, URI 등 애플리케이션 계층의 데이터를 분석하여 지능적으로 트래픽을 분산한다.

활용 예시: * /api로 들어오는 요청은 백엔드 API 서버로 라우팅

/images로 들어오는 요청은 정적 미디어 서버(CDN 등)로 라우팅

보안 관점의 이점: WAF(Web Application Firewall)와 연동하여 SQL Injection, XSS 등의 악의적인 페이로드가 포함된 HTTP 요청을 L7 단에서 사전에 필터링하고 차단할 수 있다.
