## 1. 네트워크 시작하기 (통신 모델과 아키텍처)

### 1.1. 네트워크의 규모와 구조 변화
* **홈 네트워크:** 모뎀 - 공유기(AP) - 엔드포인트(노트북, 스마트폰 등)로 연결되는 소규모 구성.
* **데이터 센터 네트워크:** 수많은 서버가 연결되어 엄청난 트래픽(10G~400G)을 처리하는 거대 인프라.
  * 과거의 전통적인 3계층(Access-Aggregation-Core) 구조에서, 최근에는 트래픽의 병목 현상을 줄이고 확장성을 높이기 위해 2계층인 **스파인-리프(Spine-Leaf) 구조**로 변화하는 추세이다.
  * 서버들은 리프(Leaf) 역할을 하는 ToR(Top of Rack) 스위치에 주로 10G/25G로 연결되고, ToR 스위치들은 최상단의 스파인(Spine) 스위치와 40G/100G 이상의 고속 대역폭으로 연결된다.

### 1.2. OSI 7계층과 TCP/IP 프로토콜 스택
현재 네트워크 물리/논리 규약의 실질적 표준은 **TCP/IP**이다. 통신의 흐름을 세분화한 OSI 7계층의 역할은 다음과 같다.

1. **물리 계층 (L1):** 전기 신호를 그대로 전달. (주소 개념 없음 / 장비: 허브, 리피터, 케이블, 트랜시버)
2. **데이터 링크 계층 (L2):** 물리적 MAC 주소 기반 통신. 에러 탐지 기능 수행. (장비: 스위치)
3. **네트워크 계층 (L3):** 논리적 IP 주소를 이해하고 목적지까지 최적의 경로(Routing) 탐색. (장비: 라우터)
4. **전송 계층 (L4):** 포트 번호를 통해 통신의 신뢰성(TCP) 보장. (장비: 로드밸런서, L4 스위치)
5. **세션 계층 (L5):** 통신 세션의 연결, 유지, 해제 담당.
6. **표현 계층 (L6):** 데이터의 변환, 압축, 암호화 수행 (MIME 인코딩 등).
7. **응용 계층 (L7):** 사용자가 체감하는 애플리케이션 서비스 (HTTP, SMTP 등).

* **캡슐화(Encapsulation):** 송신 시 상위 계층에서 하위 계층으로 내려가며 각 계층의 헤더(Header)가 덧붙여지는 과정. (수신 시에는 헤더를 벗겨내는 **디캡슐레이션**이 일어난다.)

## 2. 네트워크 연결과 구성요소 (물리적 인프라)

### 2.1. 네트워크 회선 및 통신 기술
* **인터넷 회선 vs 전용 회선:** 전용 회선은 비용이 비싸지만 대역폭이 보장됨. (저속 구간에서는 한쪽 링크 다운 시 반대쪽도 다운시켜 우회 경로를 찾게 하는 **LLCF** 기술이 불필요함).
* **VPN (Virtual Private Network):** 공용망(인터넷)을 전용선처럼 안전하게 쓰기 위한 가상 연결 기술. 특히 **MPLS VPN**은 공용 회선을 사용하여 전용선 대비 비용을 획기적으로 낮출 수 있다.
* **WDM / DWDM:** 광케이블 통신 기술. DWDM이 기존 WDM보다 훨씬 조밀하게 많은 채널(파장)을 전송할 수 있다.
* **기가 인터넷 기술:** FTTH(Fiber To The Home) 기반.
  * **PTP:** 가입자와 사업자 간 케이블 1:1 직접 연결.
  * **AON / PON:** 스위치나 광분배기를 사용하여 케이블 포설 비용을 줄이는 방식.

### 2.2. 핵심 네트워크 장비
* **NIC (네트워크 인터페이스 카드):** 컴퓨터를 네트워크에 연결. 직렬화, MAC 주소 식별, 흐름 제어를 담당한다.
* **케이블:** * **트위스티드 페어:** UTP, FTP, STP 규격 존재 (쉴드 유무 차이).
  * **광케이블:** 싱글모드(장거리)와 멀티모드(단거리/저렴함)로 구분.
* **트랜시버 (Transceiver):** 장비 내부의 전기 신호와 외부의 광/전기 신호를 상호 변환해 주는 핵심 부품.

## 3. 네트워크 통신하기 (주소 체계와 흐름)

### 3.1. 통신 방식의 분류
* **Unicast (1:1):** 가장 일반적인 1대1 통신.
* **Broadcast (1:All):** 같은 네트워크 대역 내의 모든 기기에게 전송.
* **Multicast (1:Group):** 특정 그룹에 속한 기기들에게만 전송.
* **Anycast (1:1):** 다수의 후보군 중 가장 가까운(최적의) 1대의 기기와 통신.
* *(참고: 네트워크 부하를 일으키는 BUM 트래픽 = Broadcast, Unknown Unicast, Multicast)*

### 3.2. MAC 주소 vs IP 주소
* **MAC 주소 (L2):** 하드웨어에 고정된 물리적 주소. (48비트, 16진수 12자리 / OUI(제조사) + UAA(고유번호)로 구성). NIC는 목적지 MAC 주소가 자신과 다르면 패킷을 즉시 폐기(Drop)한다.
* **IP 주소 (L3):** 사용자가 변경 가능한 논리적 주소. 네트워크 주소(동네)와 호스트 주소(집)로 나뉘는 계층적(Hierarchical) 구조를 가진다.
  * **IPv4:** 32비트 주소 체계.
  * **클래스(Class):** A, B, C, D, E 클래스로 나누어 네트워크 대역폭을 관리했던 과거 방식 (Classful). 현재는 서브넷 마스크(Subnet Mask)를 활용해 원하는 크기만큼 네트워크를 유연하게 쪼개는 클래스리스(Classless) 및 서브네팅 방식을 사용한다.
  * **NAT (Network Address Translation):** 한정된 공인 IP를 아껴 쓰고 보안을 높이기 위해, 내부망에서는 사설 IP를 쓰고 밖으로 나갈 때만 공인 IP로 변환하는 기술.

## 4. Result & Insight (오늘의 회고)
단편적으로 흩어져 있던 네트워크 장비(스위치, 라우터)와 주소 체계(MAC, IP)의 역할이 OSI 7계층이라는 큰 틀 안에서 어떻게 유기적으로 맞물려 돌아가는지 뼈대를 잡을 수 있었다. 

특히 프로젝트를 기획할 때, 엣지(Edge) 단말인 라즈베리파이(엔드포인트)에서 발생한 센서 데이터가 홈 라우터(공유기)를 거쳐 백엔드 클라우드 서버로 전달될 때, 각 구간별로 헤더(Header)가 캡슐화/디캡슐레이션되는 과정을 머릿속으로 생생하게 그릴 수 있게 되었다. 다음 주차에는 이 데이터가 탈취되지 않도록 막는 '웹 보안' 영역에 집중할 계획이다.
