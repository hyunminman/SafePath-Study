 [Frontend] React DOM 조작 최적화 및 보안 적용

작성일: 2026.07.19
주제: 엣지-클라우드 데이터 파이프라인에서 프론트엔드 성능 극대화 및 렌더링 보안

1. React의 렌더링 파이프라인과 Virtual DOM

React는 상태(State)나 속성(Props)이 변경되면 새로운 Virtual DOM 트리를 메모리상에 생성한다.

이후 이전 Virtual DOM과 새로운 Virtual DOM을 비교(Diffing) 알고리즘을 통해 대조하고, 실제 변경된 부분만 브라우저의 실제 DOM에 반영(Reconciliation)한다.

이 방식은 효율적이지만, 불필요한 상태 변경이 잦거나 컴포넌트 트리가 방대할 경우 Virtual DOM을 생성하고 비교하는 연산 자체가 병목이 될 수 있다.

2. 렌더링 성능 최적화 기법 (Memoization)

다량의 데이터를 처리할 때 리소스 낭비를 막기 위한 핵심 훅(Hooks).

React.memo: 컴포넌트가 받는 Props가 변경되지 않았다면, 리렌더링을 방지하고 이전 렌더링 결과를 그대로 재사용(캐싱)한다.

useMemo: 복잡한 연산(예: 대규모 배열 필터링, 정렬 등)의 '결과값'을 캐싱하여, 불필요한 렌더링 시마다 동일한 계산이 반복되는 것을 방지한다.

useCallback: 자식 컴포넌트에 이벤트 핸들러(함수)를 Props로 전달할 때, 부모가 렌더링될 때마다 함수 객체가 재생성되는 것을 막기 위해 '함수의 참조값' 자체를 캐싱한다.

3. 프론트엔드 단의 보안 적용 (XSS 방어)

React는 기본적으로 데이터 바인딩({data}) 시 내부적으로 문자열 이스케이프 처리를 수행하여 기본적인 XSS 공격을 방어한다.

 보안 취약 포인트 (dangerouslySetInnerHTML):

에디터 등으로 작성된 외부 HTML 문자열을 화면에 그대로 렌더링해야 할 때 이 속성을 사용하게 되는데, 악성 스크립트가 포함되어 있을 경우 그대로 실행되는 취약점이 발생한다.

해결책: 이 속성을 사용할 때는 반드시 DOMPurify 와 같은 검증된 라이브러리를 거쳐 악성 스크립트를 철저히 필터링(Sanitize)한 후 주입해야 안전하다.
